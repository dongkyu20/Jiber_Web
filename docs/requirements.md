# 요구사항 정의서

## 기능적 요구사항

| 순번 | 요구사항명 | 요구사항 상세 | 필수 여부 |
| --- | --- | --- | --- |
| F01 | 주택 실거래가 정보 수집 | 국토교통부 아파트 실거래가 공공 API 데이터를 수집하고, 서비스에서 조회할 수 있도록 가공하여 DB에 저장한다. | 필수 |
| F02 | 주택 실거래가 검색 | 사용자가 원하는 지역, 단지명, 거래유형, 면적, 금액, 거래연도 조건에 따라 주택 실거래가 검색 결과를 제공하고 지도와 상세 화면에서 확인할 수 있게 한다. | 필수 |
| F03 | 관심 지역 정보 관리 | 로그인 사용자가 관심 아파트와 관심 지역을 등록, 조회, 삭제하고 저장한 지역을 지도에서 다시 확인할 수 있게 한다. |  |
| F04 | 회원 관리 | 사용자는 회원가입을 할 수 있고, 서비스는 현재 회원의 인증 상태, 기본 정보, 권한 정보를 관리한다. | 필수 |
| F05 | 로그인 관리 | 사용자는 로그인, 로그아웃을 할 수 있고, 서비스는 refresh cookie 기반 로그인 상태 복구와 OAuth 로그인을 지원한다. | 필수 |
| F06 | 공지사항 관리 | 관리자는 공지사항을 등록, 수정, 삭제할 수 있고 모든 사용자는 커뮤니티 화면에서 공지사항 목록과 상세 내용을 조회할 수 있다. |  |
| F07 | 커뮤니티 게시글 조회 | 사용자는 커뮤니티 화면에서 공지, 자유게시판, 매매후기, Q&A 게시글 목록을 조회하고 카테고리별로 필터링할 수 있다. |  |
| F08 | 커뮤니티 게시글 상세 조회 | 사용자는 커뮤니티 게시글 상세 화면에서 게시글 본문, 작성자, 작성일, 조회수, 추천수, 관련 단지 정보, 댓글 목록을 확인할 수 있다. |  |
| F09 | 커뮤니티 댓글 입력 | 로그인 사용자는 커뮤니티 게시글 상세 화면에서 댓글을 입력할 수 있다. |  |
| F10 | 아파트 가격 추정 | 로그인 사용자는 아파트 상세 화면에서 가격 추정 결과를 조회할 수 있다. |  |
| F11 | 가격 영향 요인 설명 | 로그인 사용자는 SHAP 기반 가격 영향 요인을 차트로 확인할 수 있다. |  |
| F12 | 부동산 채팅 | 로그인 사용자는 부동산 정보와 관련된 질문을 입력하고 부동산 상세 분석 정보를 채팅 컨텍스트로 활용할 수 있다. |  |
| F13 | 관리자 화면 | 관리자는 관리자 화면에서 공지사항 등록 기능과 관리자 권한 상태를 확인할 수 있다. |  |

## 비기능적 요구사항

| 순번 | 요구사항명 | 요구사항 상세 | 필수 여부 |
| --- | --- | --- | --- |
| NF01 | 공공데이터의 정확성 | 공공데이터 원천값, 수집 이력, 정제 결과를 추적할 수 있어야 한다. |  |
| NF02 | 가용성 | 사용자는 일반 브라우저와 모바일 환경에서 주요 기능을 사용할 수 있어야 한다. |  |
| NF03 | 응답성 | 지도 검색과 조건 검색은 사용자가 체감 가능한 시간 안에 결과를 반환해야 한다. |  |
| NF04 | 사용자 편의성 | 사용자는 별도 사용법을 학습하지 않아도 검색, 상세 조회, 로그인, 관심 기능, 커뮤니티 기능을 사용할 수 있어야 한다. |  |
| NF05 | MSA REST API 적용 | 프론트엔드는 Spring Boot의 `/api/v1/**` REST API를 통해 서비스 데이터를 요청하고, model-server는 내부 API만 제공한다. |  |
| NF06 | MSA OAuth 적용 | OAuth 인증 코드, provider token, pending token은 프론트엔드 URL이나 브라우저 저장소에 노출하지 않는다. |  |
| NF07 | 토큰 보안 | access token은 메모리에만 보관하고 refresh token은 HttpOnly cookie로만 보관한다. |  |
| NF08 | 권한 보안 | 보호 API는 인증과 역할 검사를 수행하고, 사용자는 본인 소유 데이터에만 접근할 수 있어야 한다. |  |
| NF09 | 민감정보 보호 | API key, OAuth secret, JWT secret, DB password는 코드, 문서, 로그, 테스트 출력에 포함하지 않는다. |  |
| NF10 | 한국어 UI | 사용자에게 보이는 화면 문구, 버튼, 오류 메시지, 빈 상태 안내는 자연스러운 한국어로 제공한다. |  |
| NF11 | 오류 응답 표준화 | API 오류는 공통 오류 응답 형식을 따른다. |  |
| NF12 | 반응형 화면 | 주요 화면은 데스크톱과 모바일 화면에서 레이아웃이 깨지지 않아야 한다. |  |
| NF13 | 테스트 가능성 | backend, frontend, model-server는 영역별 테스트와 smoke 검증이 가능해야 한다. |  |
| NF14 | 투자 조언 제한 | 가격 추정, SHAP 설명, 채팅 답변, 커뮤니티 안내 문구는 매수/매도 추천, 투자 판단, 수익률 보장으로 표현하지 않는다. |  |
